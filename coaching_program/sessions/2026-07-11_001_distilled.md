# Session 001 — 2026-07-11 (distilled)

**Type:** Methodology design. Not client-specific. This session created the
project's first `coaching_plan.md`.

## What this session decided

- **Core philosophy:** Consistency is the delivery mechanism; no process works
  without it. Hierarchy: (1) Adherence, (2) Progression, (3) Optimization.
  Coaching is engineered for adherence, not theoretical optimality.
- **Phase 1 = Daily GM Foundation:** Client sends "GM" every morning for an
  **undisclosed** period. Undisclosed because timelines can be gamed, are audited,
  and are never "forever" (so the behavior never becomes identity); and because a
  GM is never too hard, so it filters clients. Coach replies to every GM (🫡) and
  pulls clients back gracefully after a miss ("day missed, GM today, keep going").
  Stacking rule: add ONE new thing at a time and KEEP the GM.
- **Objective graduation gate (Phase 1 → 2), all four must hold:**
  - G1 Tenure ≥ 14 days since first GM
  - G2 Consistency ≤ 2 misses in trailing 14 days (≥85%)
  - G3 Autonomy 0 prompted GMs in trailing 10 days
  - G4 Anchoring all trailing-10 GMs within ±3h of the anchor (median gm_time)
- **Two hard rules:** never advance early to be encouraging; never reveal the
  numbers to the client (thresholds are the coach's internal gauge only).

## Notable choices / rationale

- **G5 was proposed and rejected.** Original G5 = "≥1 clean recovery OR ≥30 days
  flawless." User cut it: extending the runway for a client who is performing
  perfectly does not align with the philosophy. Gate is G1–G4 only.
- Thresholds (14 / 2 / 10 / ±3h) are explicit **defaults, tunable** — change
  numbers, not logic.
- **Not building an agent right now** — this is a coaching *plan/methodology*, not
  a software agent. `physique-guru` etc. were not touched.

## Deliberately left OUT of the plan

- The graded below-gate response table (Forming / Fragile / Not landing / Ghost
  postures) — proposed but not adopted this session. Available to add later if
  wanted.
