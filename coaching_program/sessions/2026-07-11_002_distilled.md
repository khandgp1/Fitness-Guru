## What this session did

- **Reason for the rework:** the session-001 plan was structurally a verbatim
  copy of the user's source material. User approved the *language* but not the
  *format*, and asked to restructure with liberties.
- **New structure — layered by altitude** (replaces the old
  Philosophy→Phase1→Instrumentation→Gate→Rules essay order):
  - **Layer 1 — Operating Rules:** onboarding, daily loop, miss recovery, the
    advancement decision, the two hard rules (as terse imperatives). What the
    coach does; opened daily.
  - **Layer 2 — The Instrument:** tracked signals, definitions, and the G1–G4
    gate thresholds. Coach-internal; never shown to client.
  - **Layer 3 — Principles:** philosophy/hierarchy, why the timeline is
    undisclosed, why the two hard rules bind. Opened rarely.
- **New content added — "Onboarding / Day Zero"** (Layer 1): client enters with
  **no intake** (no forms/questionnaire/numbers); coach sends only the intro
  message; program begins next morning with the first GM (which sets
  `first_gm_date`); GM is the entire program until the gate clears.

## Decisions / liberties taken (all accepted by user via "Save it")

  references it as an action.

  (don't front-load complexity) — flagged as reversible if a light intake is
  ever wanted.
- **Standalone "Phase 1" heading dropped** — now implicit; the term survives
